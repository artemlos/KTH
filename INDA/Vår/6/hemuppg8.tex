\section*{Strategy to measure execution time}
\subsection*{Java just-in-time time differences}
To handle this, I am going to calculate a mean between the time it takes for JVM to produce machine code and the time it then takes to execute that code. This is a rough average, as it depends on the number of times the sorting algorithm is going to be called. As the number increases, it might be considered to lower the average.

\subsection*{Time differences between executions}
Ideally, I would test all methods in the same environment to be able to establish a more or less valid ratio. This is not feasible, so I will focus on executing methods several times, at different instances, and figure out an average.
